Project Part A: Basics (70%)

For this project, you will create the CRUD functionality for an inventory system.
You will build the following functionality:
Categories => you will require the category name (and id which will be managed via an auto incrementing primary key). Build the CRUD functionality to manage categories Items => you will be required to store the category_id, title, description, price, quantity, and sku. Category must appear as a drop-down selection for the user. Build the CRUD functionality to manage items Storefront => Build a storefront page for your site. Include a banner image and list of your items in a grid layout (Title, Category, Description, Price). Use react router to load the content for each of the 3 pages described above.
  • You will create a category entry form (/categories/create) where you will display a form to add a new product category.
  • The form will go to /categories/ where you will check to see that the category name is not empty and is not already in the database.
  • If there is an error, redisplay the form with the value you entered in the previous form along with an error message.
  • If the field is OK, enter the item in the database and redirect to /categories/.

Part 2


  • You will create a master list of categories (/categories/) where you will display the category names with an edit button beside each within a table.
  • When you click on the button, you will be brought to an edit page (/categories/{id}/edit) where all the fields in the database are pre-filled in the form.
  • Your form will go to /categories/{id} using the PATCH method.
  • You will do the same error-checking you did for adding a category.
  • If there are no errors, update the database entry with the new values.
  • Also included on the /categories/ page is a link to add a new category.

Part 3


  • You will create a product entry form (/items/create) where you will display a form to add a new item to your database.
  • Required fields are: category(id from categories), title, description, price, quantity, sku, and picture.
  • The form will go to /items/ using the POST method where you will be check to see that no field is left empty.
  • If any field is empty, redisplay the form with the values you entered in the previous form (with the exception of the picture field).
  • Display an error beside any field that is in error.
  • If the fields are ok, enter the item into the database, move the image to an images folder on your web server, and redirect to /items/.

Part 4


  • You will create a master list of products (/items/) where you will display the item's name with an edit and delete button beside each within a table.
  • This page will also contain a link to add a new item.
  • When you click on the edit button, you will be brought to an edit page (/items/{id}/edit) where all the fields in the database are pre-filled in the form.
  • Your form will go to /items/{id} using the PATCH method.
  • You will do the same error-checking you did for adding an item.
  • If there are no errors, update the database entry with the new values.
  • When you click on the delete button, you will need to confirm that you intend to delete the item (use javascript).
  • If you confirm the delete, you will go to /items/{id} using the DELETE method, soft delete the item from the database and redirect to /items/. Also, delete the image from the images folder

NOTE: Don't forget to include an export of your database
